Atmel Studio will load all the files in the project. A project may be made up of several sources files(.c files). Each file may have some dedicated function, like handling the LCD module or GSM Modem. These files are called library files and they can be reused in several other project.

After the project is opened in Atmel Studio, our next task is editing the project. That means to make the changes we need in the program.

If the program uses LCD to display its output to the user, the first thing you wanna change is some messages that program show on startup. In all my program I write my name "By Avinash" on the LCD module at startup. You may find this and change it to your name !

The Solution Explorer on the top right hand side of Atmel Studio shows all the files present in the project.

Expand the project node to see all the files that are present in the project.

They are grouped by their type

  • Dependencies
  • Output Files
  • Libraries

The most important type of files are source files. They are not grouped and are present on root level. Find a file named main.c, in most cases you need to make changes to this file only. The rest other files are supporting library files that you should not touch initially.

Once you have made the changes, its time to build the project.

Building is the process of compiling(or assembling if file is in assembly language) various source files(.c files) in the project and linking the output files (.o) to give a final executable hex file.

To build the project, select Build Solution from the Build menu. You can also use keyboard shortcut F7 to build the project.

If everything is setup correctly and your source files do not have any syntax error then you will get the following message.

It states the number of projects succeeded and the number of projects failed. Since we have only one project in our solution we should get something like this :-

If you are good at C programming language and have made no syntax error while editing the program, it should compile without any problem and you should get BUILD SUCCEEDED message as the build output.

After a successful build, the final hex file is generated which you can burn directly to the AVR MCU using a USB Programmer. This HEX file can be found at the Debug folder inside your project folder.

The extension if HEX file is .hex

About the Source Code

In computer science "source code" is a program written in a language that is easily understood by human beings. These type of language are called high level language. There are several standard high level language like C, C++, BASIC, Java, PHP etc. But most professional embedded applications are written in C or C++. The source code that you have just downloaded is written in C language.

A program written in high level language cannot be directly executed on CPU! It must first be converted into machine language.

Learning a programming language is NOT an one night affair! It takes years to get expert on them. Also some programming languages like C looks very tough to new programmers. Thus they must first learn concepts of programming with some easier languages like BASIC or LOGO.

Luckily I was born in the times when they taught programming to school kids like they teach MS Office nowadays!

Compilers

Compiler is a computer software that converts a program written high level language to machine code that can be directly executed on a CPU. So we need a compiler that converts the program written in C language to machine code for AVR series MCUs. Since compiler is a "product" thus it is developed by many companies and sold at different prices. I have selected a compiler that is also good for your pocket. I use avr-gcc for all my projects. Latest version comes bundled with Atmel Studio 6 so you don't need to install it separately.

IDE or Integrated Development Environment

Installing a compiler only is NOT enough to get started with development. Because a compiler takes as input the programs written in high level language in form of text files. You can theoretically create those program files in any text editor like Notepad under Windows. But to make the entire task manageable, one GUI software called the IDE is required to create those program files. An IDE offers much more than just editing programs.

  • Add and edit files to project.
  • The editor is much more powerful than Notepad and is specially designed for writing programs in C and thus has following advantage.
    • Your typing gets faster with the assistance of auto-complete where the editor offers you to chose long variable and function names from a list.
    • Any syntax error is underlined RED in real time thus allowing to to make corrections.
    • Better view of the program. Thanks to syntax highlight feature different parts of the program gets different colours and looks.
  • Compiler also manages the compiler settings which are very crucial for proper and error free compilation of the programs.
  • It calls the compiler for your from just a click of menu.
  • Shows the result of the compilation process.
  • It also helps in debugging of the programs.
